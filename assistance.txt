                              PROJECT SUMMARY
================================================================================

PROJECT NAME: ACACONNECT
TYPE: Full-stack Event Management System for Academic Institutions
CURRENT STATUS: Phase 2.5 - Mock Payment System Integrated
DEVELOPMENT PERIOD: January 2026
COMPLETION LEVEL: ~85% (Core features complete, advanced features pending)

TECH STACK:
- Frontend: React.js with React Router
- Backend: Node.js + Express.js
- Database: MongoDB with Mongoose ODM
- ML Service: Python Flask with KNN algorithm (10 lakh dataset)
- Chatbot: Python Flask + Ollama + ChromaDB (RAG pipeline)
- Authentication: JWT-based with bcrypt password hashing
- Email: Nodemailer with Gmail SMTP
- File Upload: Multer for event cover photos

================================================================================
                              CURRENT WORKFLOW
================================================================================

EVENT LIFECYCLE:
1. Event Team creates event with comprehensive details
2. Event submitted for approval (FSM: DRAFT → SUBMITTED → UNDER_REVIEW)
3. Treasurer reviews and approves budget (UNDER_REVIEW → TREASURER_APPROVED)
4. General Secretary reviews and can edit (TREASURER_APPROVED → GENSEC_APPROVED)
5. Chairperson gives final approval (GENSEC_APPROVED → CHAIRPERSON_APPROVED → PUBLISHED)
6. Published events visible to participants
7. Participants can register and make payments
8. Registration status tracked throughout

PARTICIPANT JOURNEY:
1. Signup with email OTP verification
2. Login to ParticipantHomePage
3. Browse events in Events tab
4. Use Find Events for ML-powered recommendations
5. Register for events (free or paid)
6. Complete payment if required
7. View registrations in My Registrations tab
8. Complete pending payments by clicking PENDING status

================================================================================
                              STARTUP INSTRUCTIONS
================================================================================

PREREQUISITES:
1. Node.js and npm installed
2. MongoDB running on localhost:27017
3. Python 3.x for ML and chatbot services
4. Ollama installed with Mistral model (for chatbot)

STARTUP SEQUENCE:
1. Start MongoDB service
2. Backend: cd backend && npm start (port 5000)
3. Frontend: cd frontend && npm start (port 3000)
4. ML Service: cd ml-service && python app.py (port 5001)
5. Chatbot Service: cd chatbot-service && python app.py (port 5002)

OR use START_ALL.bat to start all services at once

ENVIRONMENT SETUP:
- backend/.env contains MongoDB URI, JWT secret, email credentials
- Gmail app password configured for OTP emails
- All dependencies installed via npm/pip
