PROJECT ABSTRACT:
Workflow-driven College Event Management System that automates the entire event lifecycle:
- Event planning, approval, execution, and post-event activities
- Role-based access control (administrators, event teams, logistics, finance, students)
- Budget consolidation using MapReduce-based aggregation
- ML-based budget prediction module
- ML-based event suggestion system
- AI chatbot for event assistance
- GPS/geofencing-based attendance verification
- Automated PDF certificate generation
- Modular, layered architecture for scalability

TECHNICAL REQUIREMENTS:

APPLICATION TYPE:
- Full-stack web application (Backend API + Frontend UI)
- Web browser-based (laptop + mobile browsers)
- Cross-platform by default

TECH STACK:
- Frontend: React.js
- Backend: Node.js with Express.js
- Database: MongoDB with Mongoose ODM (MIGRATED FROM POSTGRESQL)
- AI/ML: Python (Scikit-learn), Flask/FastAPI for ML APIs
- Authentication: JWT-based
- File Storage: Server/cloud storage
- Location: HTML5 Geolocation API

USER ROLES & PERMISSIONS:
1. Admin: Full access to all system functions
2. Chairperson: Final event approval, announcements
3. General Secretary: View all workflows (read-only oversight)
4. Treasurer: Budget aggregation & ML prediction access
5. Event Team: Create events, submit for approval
6. Logistics: Add resource requirements to events
7. HR: Volunteer assignment and management
8. Hospitality: Room allocation and management
9. Students: Register for events, mark attendance, download certificates

WORKFLOWS:
1. Event Team: Login → Create event → Submit for approval → Track status
2. Logistics/HR/Hospitality: Login → View assigned events → Add requirements → Submit
3. Treasurer: Login → View consolidated budget → ML prediction → Approve/reject
4. Chairperson: Login → View final details → Approve/reject → Announcements
5. Students: Login/Signup → View events → Register → Attendance → Certificate
6. General Secretary: Login → View all workflows (monitoring)
7. Admin: Full access

EVENT LIFECYCLE FLOW (ENFORCED):
Event Creation (Event Team) → Submit for Approval → Treasurer Approval → General Secretary Approval (with edit capability) → Chairperson Final Approval → Event Published → Requirements Sent to Teams (HR, Logistics, Hospitality) → Student Registration → Attendance (GPS) → Certificate Generation

NEW WORKFLOW STATUSES:
- CREATED: Event created by Event Team
- PENDING_TREASURER: Submitted for treasurer review
- TREASURER_APPROVED: Approved by treasurer, sent to Gen Sec
- PENDING_GEN_SEC: Awaiting General Secretary review
- GEN_SEC_APPROVED: Approved by Gen Sec, sent to Chairperson
- PENDING_CHAIRPERSON: Awaiting final approval
- CHAIRPERSON_APPROVED: Final approval granted
- PUBLISHED: Event published and visible to students
- REJECTED: Event rejected at any stage

MODULE SPECIFICATIONS:

1. AUTHENTICATION & AUTHORIZATION
   - JWT login with role extraction
   - Route protection middleware
   - APIs: POST /auth/login, POST /auth/register, GET /auth/me

2. EVENT MANAGEMENT
   - Event Team only creation
   - Fields: Title, Type, Date, Duration, Venue, Expected participants
   - Status: CREATED → BUDGET_PENDING → APPROVED → PUBLISHED
   - APIs: POST /events, PUT /events/:id/status, GET /events

3. REQUIREMENT COLLECTION
   - Logistics: equipment, food
   - HR: volunteers
   - Hospitality: rooms
   - Stored separately, linked to event

4. BUDGET AGGREGATION (MapReduce)
   - Combine similar requirements
   - Sum costs using extended MapReduce pattern
   - Output: Consolidated budget proposal

5. BUDGET PREDICTION (ML)
   - Model: Multiple Linear Regression
   - Input: Participants, Event type, Duration, Venue cost, Resource cost
   - Flow: Backend → ML API → Prediction → Treasurer Dashboard

6. TREASURER APPROVAL
   - Compare aggregated vs ML predicted budget
   - Approve/Reject decision

7. STUDENT REGISTRATION
   - Post-publication registration
   - Paid/Free support
   - Registration status tracking

8. EVENT SUGGESTION (ML)
   - Model: KNN recommendation system
   - Data: User-event interactions
   - Output: Personalized event list

9. AI CHATBOT
   - Capabilities: Event info, Registration help, Certificate queries
   - Flow: User → Intent Detection → Backend API → Response

10. LOCATION-BASED ATTENDANCE
    - GPS capture → Haversine validation (100m radius)
    - Time-bound attendance marking
    - Attendance stored for certificate eligibility

11. CERTIFICATE AUTOMATION
    - Rule: attendance = TRUE required
    - PDF generation using PDFKit
    - Template-based with Name, Event, Date

12. NOTIFICATIONS
    - Triggers: Event approved, Registration confirmed, Certificate ready
    - Channels: In-app + Email

HYBRID KNN RECOMMENDATION SYSTEM - January 2026
=====================================

✅ UPGRADED TO HYBRID KNN WITH PATTERN LEARNING
✅ System now uses 10 lakh dataset to learn patterns and apply to published events
✅ Implementation Details:
   - Pre-trained KNN model on 1,048,575 events (10 lakh dataset)
   - Finds 50 nearest neighbors from dataset to learn interest patterns
   - Creates learned pattern vector (average of 50 similar events)
   - Scores YOUR published events using hybrid approach
   - Hybrid Score = 70% direct similarity + 30% pattern similarity
✅ ML Service Enhancements:
   - /recommend-knn endpoint: Hybrid KNN with pattern learning
   - Uses sklearn NearestNeighbors with cosine metric
   - Learns from 10 lakh events, recommends from YOUR events only
   - Returns top k=5 recommendations with similarity scores
✅ Backend Integration:
   - mlController.js updated to call /recommend-knn endpoint
   - Sends all published events to ML service for scoring
   - Receives ML-powered similarity scores for each event
   - Sorts by hybrid score and returns top 5 recommendations
✅ Frontend Display:
   - Ranked display with #1, #2, #3, etc. badges
   - Match Score percentage (e.g., 85.3%)
   - Breakdown showing Direct match % and Pattern match %
   - Professional UI with gradient badges and score details
✅ How It Works:
   1. User selects interests (e.g., Programming, Database, DSA)
   2. ML service finds 50 similar events from 10 lakh dataset
   3. Creates learned pattern from those 50 events
   4. Scores each published event:
      - Direct similarity: User interests vs Event tags
      - Pattern similarity: Learned pattern vs Event tags
   5. Combines scores: 70% direct + 30% pattern
   6. Returns top 5 ranked events with scores
✅ Benefits:
   - Learns correlations from 1 million events (e.g., SQL relates to Programming)
   - Smarter recommendations than simple tag matching
   - Discovers non-obvious event suggestions
   - Uses real ML patterns while recommending YOUR events
   - No useless recommendations from training dataset

MATHEMATICAL APPROACH:
- Feature Space: 16-dimensional binary vectors (one per tag)
- Similarity Metric: Cosine similarity
- Pattern Learning: Average vector of k=50 nearest neighbors
- Hybrid Scoring: Weighted combination of direct and pattern similarities
- Ranking: Sort by hybrid score, return top k=5

TECHNICAL STACK:
- ML Service: Flask + scikit-learn + pandas + numpy
- Model: KNN with cosine similarity (k=50 for pattern learning, k=5 for recommendations)
- Dataset: 1,048,575 events with 16 features
- Backend: Node.js + Express + axios
- Frontend: React with ranked display UI
