ISSUE: All Events Showing 28% Score
====================================

PROBLEM:
--------
When using CF or Hybrid recommendations, all 10 events show the same 28% score.
This indicates the fallback method is being used instead of the ML service.

ROOT CAUSE:
-----------
The frontend is passing full event objects to the ML endpoints, but the ML service
expects events in format: {id: "...", tags: [...]}

The backend mlController should be formatting events correctly, but it's not being
called properly or the ML service is returning errors.

QUICK FIX:
----------
The issue is in ParticipantHomePage.jsx line 408-420.

When calling CF or Hybrid, it passes:
  events: events  // Full event objects

But ML service expects:
  events: events.map(e => ({id: e._id, tags: e.tags}))

SOLUTION:
---------
Update ParticipantHomePage.jsx suggestEvents function:

Change line 408 from:
  events: events,

To:
  events: events.map(e => ({id: e._id.toString(), tags: e.tags || [], title: e.title})),

And line 416 from:
  events: events,

To:
  events: events.map(e => ({id: e._id.toString(), tags: e.tags || [], title: e.title})),

This will properly format events for the ML service.

ALSO:
-----
Change k: 10 to k: 5 on lines 410 and 418 to limit to top 5 recommendations.

EXPECTED RESULT AFTER FIX:
--------------------------
- Varied scores (not all 28%)
- Top 5 events only (not 10)
- CF scores > 0% with real collaborative patterns
- Hybrid showing combined KNN + CF scores

The 28% score is coming from: 2 matching tags / 7 total unique tags = 0.285 = 28.5%
This is the Jaccard similarity fallback, not the real CF algorithm.
