DASHBOARD IMPLEMENTATION GUIDE
=====================================

This document outlines the standard structure and implementation patterns for creating consistent dashboards in the ACA Connect system.

## DASHBOARD STRUCTURE
=====================

### 1. OVERVIEW SECTION (Required)
- **Purpose**: Main landing page with stats and action buttons
- **Components**:
  - Stats cards showing relevant metrics
  - Action buttons to navigate to different sections
  - Clean card-grid layout

**Example Structure**:
```jsx
const renderOverview = () => (
  <div className="card-grid fade-in">
    <div className="card">
      <h3 className="card-title">Dashboard Name</h3>
      <div className="card-content">
        <p><strong>Stat 1:</strong> {stats.value1}</p>
        <p><strong>Stat 2:</strong> {stats.value2}</p>
      </div>
    </div>
    <div className="card">
      <h3 className="card-title">Actions</h3>
      <div className="card-content">
        <button className="btn btn-primary" onClick={() => setActiveView('section1')}>
          Action 1
        </button>
        <button className="btn btn-secondary" onClick={() => setActiveView('section2')}>
          Action 2
        </button>
      </div>
    </div>
  </div>
);
```

### 2. NAVIGATION PATTERN
- **State Management**: Use `activeView` state to control sections
- **Back Navigation**: Always provide "Back to Overview" functionality
- **Consistent Styling**: Use standard nav-header pattern

**Standard Navigation Header**:
```jsx
<div className="nav-header">
  <h3 className="nav-title">Section Name</h3>
  <button 
    onClick={() => setActiveView('overview')}
    style={{ 
      background: 'none', 
      border: 'none', 
      color: '#00E5FF', 
      fontSize: '0.85rem', 
      cursor: 'pointer',
      padding: '0.25rem 0',
      pointerEvents: 'auto',
      zIndex: 999999,  // CRITICAL: Prevents click blocking
      position: 'relative'
    }}
  >
    ← Back to Overview
  </button>
</div>
```

### 3. COMMON SECTIONS
- **Overview**: Main dashboard with stats
- **List View**: Table or grid of items
- **Detail View**: Individual item details
- **Form View**: Create/edit functionality

## TECHNICAL IMPLEMENTATION
==========================

### 1. COMPONENT STRUCTURE
```jsx
export default function RoleDashboard() {
  const [activeView, setActiveView] = useState('overview');
  const [data, setData] = useState([]);
  const [selectedItem, setSelectedItem] = useState(null);

  const handleBackToOverview = () => {
    setActiveView('overview');
    setSelectedItem(null);
  };

  return (
    <div>
      {activeView === 'overview' && renderOverview()}
      {activeView === 'section1' && renderSection1()}
      {activeView === 'section2' && renderSection2()}
    </div>
  );
}
```

### 2. CRITICAL CSS FIXES
- **Z-Index Issue**: Back buttons need `zIndex: 999999` to prevent click blocking
- **Animation Conflicts**: Remove `fade-in` class if buttons become unclickable
- **Position**: Use `position: 'relative'` for proper layering

### 3. API INTEGRATION
- **Fetch Data**: Load data on component mount
- **Error Handling**: Always include try-catch blocks
- **User Feedback**: Use alerts for success/error messages
- **Refresh Data**: Reload after CRUD operations

**Standard API Pattern**:
```jsx
const fetchData = async () => {
  try {
    const response = await axios.get('/api/endpoint');
    setData(response.data);
  } catch (error) {
    console.error('Failed to fetch data:', error);
  }
};

const handleAction = async (id) => {
  try {
    await axios.post(`/api/endpoint/${id}`);
    alert('Action completed successfully!');
    fetchData(); // Refresh data
  } catch (error) {
    alert('Failed to complete action: ' + (error.response?.data?.message || 'Unknown error'));
  }
};
```

## STYLING STANDARDS
===================

### 1. BUTTON CLASSES
- `btn btn-primary`: Main actions (blue gradient)
- `btn btn-secondary`: Secondary actions (cyan border)
- `btn btn-success`: Positive actions (green)
- `btn btn-danger`: Destructive actions (red)

### 2. CARD LAYOUTS
- Use `card-grid` for overview sections
- Use `events-grid` for item listings
- Use `card` class for individual containers

### 3. STATUS BADGES
- `status-badge success`: Completed items
- `status-badge status-pending`: Pending items
- `status-badge status-approved`: Approved items
- `status-badge status-rejected`: Rejected items

## COMMON ISSUES & SOLUTIONS
============================

### 1. UNCLICKABLE BUTTONS
**Problem**: Buttons become unclickable after animations
**Solution**: Add `zIndex: 999999` and `position: 'relative'`

### 2. NAVIGATION ISSUES
**Problem**: Back buttons don't work consistently
**Solution**: Use dedicated `handleBackToOverview` function

### 3. DATA REFRESH
**Problem**: Data doesn't update after actions
**Solution**: Call `fetchData()` after successful operations

### 4. FORM HANDLING
**Problem**: Complex forms with file uploads
**Solution**: Use FormData for multipart requests
