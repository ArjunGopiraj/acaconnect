BACKEND FILE STRUCTURE - QUICK REFERENCE
========================================

📁 src/config/
  └─ db.js - MongoDB connection setup using Mongoose

📁 src/controllers/
  ├─ admin.controller.js - Admin dashboard stats, user management, event management
  ├─ approval.controller.js - Event approval workflow logic
  ├─ budget.controller.js - Budget aggregation and management
  ├─ event.controller.js - Event CRUD operations (create, read, update, delete, publish)
  ├─ mlController.js - ML recommendation API integration (calls Flask ML service)
  └─ requirement.controller.js - Event requirements management (volunteers, rooms, etc.)

📁 src/middleware/
  ├─ auth.middleware.js - JWT token verification, user authentication
  ├─ fsm.middleware.js - Finite State Machine validation for event status transitions
  ├─ role.middleware.js - Role-based access control (RBAC) checks
  └─ workflow.middleware.js - Event workflow state validation

📁 src/models/
  ├─ Budget.js - Budget schema (aggregated_amount, predicted_amount, treasurer_status)
  ├─ EventRequirement.js - Event requirements schema (volunteers, rooms, refreshments)
  ├─ Events.js - Main event schema (title, type, date, status, tags, cover_photo, etc.)
  ├─ EventType.js - Event type definitions (Technical, Non-Technical, Hackathon, etc.)
  ├─ Notification.js - Notification schema (type, message, user, read status)
  ├─ Role.js - User role definitions (ADMIN, EVENT_TEAM, TREASURER, etc.)
  └─ User.js - User schema (name, email, password, role_id)

📁 src/routes/
  ├─ admin.routes.js - Admin endpoints (/admin/stats, /admin/users, /admin/events)
  ├─ auth.routes.js - Authentication endpoints (/auth/login, /auth/register)
  ├─ budget.routes.js - Budget endpoints (/budgets/aggregate, /budgets/predict)
  ├─ event.routes.js - Event endpoints (/events/create, /events/publish, /events/approve)
  ├─ ml.js - ML recommendation endpoints (/ml/recommend)
  ├─ notification.routes.js - Notification endpoints (/notifications/user/:userId)
  └─ requirement.routes.js - Requirement endpoints (/requirements/event/:eventId)

📁 src/services/
  ├─ budgetAggregation.service.js - MapReduce-based budget aggregation logic
  ├─ fsm.service.js - Finite State Machine for event workflow (DRAFT → PUBLISHED)
  └─ notification.service.js - Notification creation and delivery logic

📁 src/uploads/events/
  └─ Event cover photos (jpg, png files uploaded by Event Team)

📁 src/utils/
  └─ constants.js - Application constants (EVENT_STATUS, ROLES, etc.)

📄 Root Scripts (src/):
  ├─ server.js - Main Express server entry point (port 5000)
  ├─ seedRoles.js - Seeds 9 roles into database
  ├─ seedUsers.js - Seeds test users for all roles
  ├─ seedEventTypes.js - Seeds default event types
  ├─ seedEvents.js - Seeds 10 published events with cover photos
  ├─ checkFSM.js - Tests FSM state transitions
  ├─ cleanupEvents.js - Deletes all events from database
  ├─ quickStats.js - Shows quick database statistics
  ├─ testEventCreation.js - Tests event creation workflow
  ├─ testFSM.js - Tests FSM service functionality
  └─ viewDB.js - Views database contents

📄 Root Files (backend/):
  ├─ .env - Environment variables (MONGODB_URI, JWT_SECRET)
  ├─ package.json - Dependencies and npm scripts
  └─ checkEventCount.js - Checks total events in database

========================================
WORKFLOW FLOW:
1. server.js → Connects to MongoDB → Starts Express server
2. Routes → Controllers → Services → Models → Database
3. Middleware validates authentication, roles, and state transitions

KEY ENDPOINTS:
- POST /auth/login - User login
- POST /events/create - Create event (Event Team)
- POST /events/:id/submit - Submit for approval
- POST /events/:id/treasurer-approve - Treasurer approval
- POST /events/:id/gensec-approve - General Secretary approval
- POST /events/:id/chairperson-approve - Chairperson approval & publish
- GET /events/published - Get all published events
- POST /ml/recommend - Get ML-powered event recommendations

DATABASE: MongoDB (college_events)
PORT: 5000
